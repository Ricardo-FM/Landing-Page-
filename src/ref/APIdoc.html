2. Response

The system response is in JSON format. Field Result will contain one of the four possible results: "Accepted", "Rejected", "Duplicate" or "Errors".

2.1. Sample Result:Accepted

Result "Accepted" means that we sold the lead at the price equal to or higher than requested in MinimumPrice field. Exact sales price will be specified in Price field.

In order to complete the sale, consumer must be redirected to the URL provided in the RedirectURL field.

Code
Copy code
{
  "Result": "1",
  "CampaignID": "123456",
  "LeadID": "D7B79F03-0001-00C0-A33E-64FF04E9C000",
  "Price": "100.00",
  "RedirectURL": "http://www.loanmatchingservice.com/?lid=123ABC"
}
2.2. Sample Result:Rejected

Result "2" or "Rejected" means that we were unable to sell the lead at the price requested in MinimumPrice field. If the value specified in MinimumPrice field is greater than 0, you may try to repost the lead with lower MinimumPrice field. For reposting instructions see Section 3.

Code
Copy code
{
  "Result": "2",
  "CampaignID": "123456",
  "LeadID": "D7B79F03-0001-00C0-A33E-64FF04E9C000",
  "Messages": {
    "Message": "Try lower minimum price"
  }
}
2.3. Sample Result:Duplicate

Result "Duplicate" means that we have previously processed the lead and it was either declined by lenders or customer has an outstanding loan. Please, do not attempt to repost the lead.

Code
Copy code
{
  "Result": "Duplicate",
  "Messages": {
    "Message": "Duplicate"
  }
}

3. Reposting
3.1. Sample Result:Rejected

If you received a response Result:Rejected you may repost the same lead with lower MinimumPrice value. Lead should be reposted to the same URL as an initial post. Response to REPOST of the lead will be in the same format as initial lead post response (see Section 2).

{%%show_bid_reposting_json%%}
4. Testing

Before your campaign can be switched to production, we need to receive a successful test post. Your account manager will be notified when successful post is received.

In order to submit test post, within the tag TestResult you need to pass one of the two possible values: "Accepted" or "Rejected". The system will respond to the test post with Result; tag containing the value posted in TestResult tag.
